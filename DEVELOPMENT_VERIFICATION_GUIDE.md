# 開發與交叉驗證指引（可遷移）

本指引用於確保「功能有真的做上去、沒做壞舊功能、可持續演進」。
適用於任何前端工具/編輯器型專案，不限本專案。

## 0. 使用規則（每次增修都必做）

1. 每一個需求都要先寫「可驗收結果」，再開始改程式。
2. 每次改動都要跑完「最小驗證閉環」：`語法 -> 功能 -> 回歸 -> 存取/匯出`。
3. 任何 bug 修正都要先定位根因，再修；禁止只改表象。
4. 每次完成後要更新本文件：
   - 新增更有效的驗證流程。
   - 刪除無效或重複流程。

## 1. 需求落地格式（先寫這段）

每個需求用以下格式記錄（可貼在 PR/commit 註解）：

```md
## 需求
- 做什麼：
- 不做什麼：

## 驗收標準
- [ ] 操作 A -> 結果 B
- [ ] 操作 C -> 結果 D

## 影響範圍
- UI:
- Runtime:
- Timeline:
- Save/Load:
- Export:
```

## 2. 最小驗證閉環（必跑）

1. 靜態/語法檢查  
   - 例如：`node --check app.js`
2. 主流程手測（針對本次需求）
3. 回歸手測（至少覆蓋相鄰功能）
4. 存檔/讀檔一致性  
   - 存檔 -> 重開 -> 讀檔 -> 行為一致
5. 匯出一致性（若功能有輸出）  
   - JSON/Binary 匯出內容與 UI/Runtime 行為一致

## 3. 交叉驗證方法（避免「看起來有改，實際沒改」）

1. 三層對照
   - UI 顯示值
   - Runtime 實際求解/運算結果
   - 匯出資料內容
2. 同一操作做雙向驗證
   - 由 UI 改值看 Runtime
   - 由資料載入看 UI 是否還原
3. 特別檢查邊界值
   - `0`、`-1`、空陣列、單元素、最大/最小值
4. 排序/順序類功能必測
   - 移動前後順序
   - 重開後順序
   - 匯出後順序

## 4. 本專案已驗證有效的重點流程（保留）

1. `order` 欄位驗證不能用 `Number(x) || fallback`  
   - `0` 會被誤判成 fallback，造成排序/求解錯亂。  
   - 改用 `Number.isFinite(...) ? value : fallback`。
2. 約束類功能要統一排序來源  
   - `getActive*`、`refresh*`、Runtime solver、Export 都要同一規則。
3. UI 清單與實際資料排序要同步  
   - 先排序，再用物件參考回填 selected index，避免選取漂移。
4. 任何「可保存功能」都要做持久化回歸  
   - 存檔與讀檔規則必須與 runtime 相同。

## 5. 回歸測試矩陣（每次至少勾一輪）

- [ ] 新功能主流程可操作
- [ ] 相關舊功能未退化
- [ ] 快捷鍵仍可用（若有）
- [ ] Timeline key 插入/刪除/拖曳正常（若有）
- [ ] Pose/Edit mode 切換後狀態正確（若有）
- [ ] 存檔/讀檔後結果一致
- [ ] 匯出後結構與行為一致

## 6. 失敗排查順序（固定）

1. 先確認資料是否真的寫入（state/model）。
2. 再確認 UI 是否正確讀取該資料。
3. 再確認 runtime 是否使用同一份資料。
4. 最後確認匯出是否用同一套轉換規則。

不要一開始就改 CSS/顯示層去掩蓋 runtime 問題。

## 7. 文件演進規範（強制）

每次增修完成，必須在此節追加一筆：

```md
### YYYY-MM-DD - 主題
- 新增流程：
- 刪除流程：
- 原因：
- 成效：
```

若某流程連續 3 次沒有抓到問題、且有更短更準確流程可替代，即刪除舊流程。

### 2026-02-23 - 約束排序與一致性驗證
- 新增流程：
- 任何 `order`/排序欄位都要檢查是否誤用 `||` 造成 `0` 失效。
- 約束類功能必做「UI順序、Runtime求解順序、Export順序」三向對照。
- 變更 `refresh*` 或 `getActive*` 後，必測「選取項是否漂移」。
- 新增/刪除/上下移動後，需重開專案再驗證順序是否保持。
- 刪除流程：
- 僅看畫面結果判定功能完成（未對照資料層與匯出層）的做法。
- 原因：
- 曾出現畫面可用但 runtime 與 export 順序不一致，且 `order=0` 被覆蓋的根因問題。
- 成效：
- 可快速定位排序錯亂是否來自資料解析、UI排序或匯出，並避免回歸。

## 8. 完工定義（DoD）

符合以下才算完成：

1. 驗收標準全過。
2. 最小驗證閉環全過。
3. 回歸矩陣已勾選。
4. 文件演進紀錄已更新。
